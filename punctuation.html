Appendix: Punctuation

Ampersands

An ampersand (U＋0026) in the context of…

A message expression
Converts a Proc object to a block, passing it to the receiver.
A parameter list
Signifies that the following identifier corresponds to the block sent along with the message.
An operator
Performs bitwise AND. The operator consisting of two consecutive ampersands performs Boolean AND.

Apostrophes

An apostrophe (U＋0027) delimits single-quoted strings.

Asterisks

An asterisk (U＋002A) in the context of…

A message expression
Explodes an object into its constituent elements.
A parameter list
Prefixes a parameter which may receive a variable number of arguments.
An operator
Performs multiplication—when alone—or exponentiation—when preceded by another asterisk.

Circumflex Accents

The circumflex accent (U＋005E) is an binary operator that performs bitwise exclusive or.

Colons

A colon (U＋003A) in the context of…

The beginning of an expression
Creates a symbol from the following identifier or string. (Unless followed by another colon).
An operator
Separates the then clause from the else clause in a ternary expression.
An operator inside a Hash literal
Separates Symbol keys from their corresponding values.
A constant reference expression
Two consecutive colons serve as the scope operator when the character to their immediate right is an uppercase US-ASCII letter: either separating one constant’s name from another, or at the beginning of such an expression.
A message expression
Two consecutive colons followed by a character other than an uppercase US-ASCII letter may separate the receiver from the selector.

Commas

A comma (U＋002C) separates expressions in a list. It is used in message expressions, parameter lists, the lvalues and rvalues of assignment expressions, Array and Hash literals, and the list of method names passed to undef.

Commercial At Signs

When the commercial at sign (U＋0040) prefixes an identifier it is a sigil introducing an instance variable—if it appears by itself—or a class variable—if it is preceded by another. In a symbol literal or method name, an @ suffix distinguishes a unary operator from a binary operator with the same name.

Curly Brackets

A pair of curly brackets (U＋007B, U＋007D) in the context of a message expression delimit a block which is sent along with the message, otherwise they delimit a hash literal.

Dollar Signs

A dollar sign (U＋0024) followed by an identifier is a sigil denoting a global variable.

Equals Signs

An equals sign (U＋003D) in the context of…

An assignment expression
Separates the variables being assigned to from the values being assigned to them.
An operator
Tests for equality if doubled, case equality if tripled, and inequality if prefixed with less‐than (U＋003C) or greater‐than (U＋003E) signs; performs pattern matching if followed by a tilde (U＋007E).
A parameter list
Signifies that a parameter is optional and its default value
Hash literals
When followed by a greater‐than sign (U＋003E) separates a key from its value.
Embedded documentation
Introduces the beginning of embedded documentation when appearing at the start of a line followed by begin, and the end when appearing at the start of a line followed by end

Exclamation Marks

An exclamation mark (U＋0021) in the context of…

A selector
Signifies the message deserves extra attention
An operator
Negates its operand, when used as a unary operator, the sense of a pattern match, when followed by a tilde (U＋007E), or a test for equality, when followed by an equals sign (U＋003D).

Full Stops

A full stop (U＋002E) is used to separate in the context of…

A message expression
It separates the receiver from the selector
A range literal
Two or three consecutive periods separate the start value from the end value
A float literal
It is the radix point separating the whole part of the number from the fractional part.
A flip-flop operator
It separates the first value from the last

Grave Accents

Grave accents (U＋0060) delimit a double‐quoted string interpreted as a command to be executed.

Greater-Than Signs

When used as a binary operator, a greater‐than sign (U＋003E) represents the corresponding inequality. When immediately followed by an equals sign, it is a binary operator representing the greater-than-or-equal-to inequality. In a Hash literal, it is preceded by an equals sign to separate keys from their values.

Hyphen Minus Signs

A hyphen minus sign (U＋002D) performs negation when used as a unary operator, and subtraction when used as a binary operator. Both are operator methods, whose selectors are distinguished by the former having a commercial at sign prefix. When followed by a greater‐than sign (U＋003E) it may represent a lambda literal.

Less-Than Signs

A less‐than sign (U＋003C) in the context of…

An operator
Tests for the corresponding inequality when solitary or followed by an equals sign (U＋003D). Serves as the spaceship operator when followed by an equals sign and greater‐than sign (U＋003E).
Class definition
Specifies the parent of the class being defined.
Singleton class definition
Two consecutive less‐than signs following the class keyword opens the singleton class of the following expression.

Low Lines

A low line (U＋005F) is used to separate digits in numeric literals, and is a valid character in identifiers.

Newlines

A newline is one of the following character sequences:

  • U＋000A (line feed)
  • 〈U＋000D, U＋000A〉 (carriage return followed by line feed)

The Statement Terminators section explains how the role of newlines in Ruby syntax is dependent on their surrounding context.

Number Signs

A number sign (U＋0023) inside a double‐quoted string literal introduces an interpolation sequence. Outside of a string literal it introduces a comment.

Parentheses

A balanced pair of parentheses (<U＋0028, U＋0029>) group arguments in message expressions, enforce precedence in compound expressions, and enclose the parameter list of a method definition.

Percent Signs

A percent sign (U＋0025) is an operator and part of a generalised quoting form for many object literals.

Plus Signs

A plus sign (U＋002B) used a binary operator, usually sums of concatenates its operands; when used as a unary operator it usually makes its operand positive. Both are operator methods, whose selectors are distinguished by the former having a commercial at sign prefix.

Question Marks

A question mark (U＋003F) suffixing a selector suggests the message is a predicate; as part of a ternary expression it separates the conditional from the first clause; immediately preceding a character, or a character escape evaluating to the same, it evaluates to that character.

Quotation Marks

A pair of quotation marks (U＋0022) delimit a double-quoted string literal.

Reverse Solidi

A reverse solidus (U＋005C) in the context of a string literal introduces an escape sequence. Immediately before a literal newline it escapes the line break.

Semicolons

A semicolon (U＋003B) is an explicit statement terminator.

Solidi

A solidus (U＋002F) is an operator and, when appearing as part of a pair, delimits a regexp literal.

Square Brackets

A pair of square brackets (<U＋005B, U＋005D>) delimit an array literal. They delimit the argument list of a message expression when its selector is a pair of square brackets which are optionally followed by an equals sign. Correspondingly, both of these selectors ([] and []=) are also valid method names.

Tildes

A tilde (U＋007E) performs bitwise complement when used as a unary operator. When used as a binary operator, it performs pattern matching if preceded by an equals sign or inverted pattern matching if preceded by an exclamation mark.

Vertical Lines

A vertical line (U＋007C is a bitwise-OR operator when used singly, a Boolean OR operator when followed by another, and serves to delimit the parameter list of a block.
